01 Introduction to Arrays Definition · Syntax · Declaration ▾

What is an Array?

Array = Multiple values of the same datatype stored in contiguous memory locations.

Problem without arrays: Storing 5 numbers requires 5 variables. Storing 10⁵ numbers? Impossible with individual variables!

Syntax:

int ar[5]; // 5-element int array
float fr[4]; // 4-element float array

Alternative Syntax (with init):

// Alt Syntax 1:
int ar[5] = {10, 20, 30, 40, 50};

// Alt Syntax 2 (size auto-detected):
int ar[] = {10, 30, 40};

Default Values

Arrays in functions (local): NOT initialized → random/garbage values

Arrays global: Automatically set to default → int=0, float=0.0, char='\0'

02 Indexing Access · Bounds · name[index] ▾

Indexing Rule

To access any cell: name[index]

For int ar[N]: valid indices are 0 to N-1

First element = ar[0]    Last element = ar[N-1]

ar[0] = 10;  ar[1] = 20;  ar[2] = 30;  ar[3] = 40;  ar[4] = 50;
// Each cell is like a variable — can print/access/read/input
cout << ar[0] + ar[3]; // 50
ar[0] = 100; // modifies first element

⚠ Out-of-Bounds Access

In C++, accessing ar[N] or ar[-1] is NOT an error — it gives a random/garbage value from adjacent memory. This is a silent bug!

int ar[3]; ar[0]=10; ar[1]=20; ar[2]=30;
cout << ar[3]; // random value — accessing out of bounds!
cout << ar[-1]; // random value — accessing out of bounds!

Arrays are always passed by reference in C++. The function receives a pointer to the same memory — any changes inside the function affect the original array!

05 Max & Min in Array INT_MIN · INT_MAX · Initialization ▾

Initialization Rule — CRITICAL

For Sum → initialize with 0 (won't affect addition)

For Product → initialize with 1 (won't affect multiplication)

For Max → initialize with INT_MIN (smallest int, any element will be larger)

For Min → initialize with INT_MAX (largest int, any element will be smaller)

⚠ Do NOT initialize max with 0 — if all values are negative, max will never update!

Max Element — TC: O(N), SC: O(1)

#include <climits>

int maxArray(int ar[], int N){
    int mx = INT_MIN;
    for(int i=0; i<N; i++){
        if(ar[i] > mx){
            mx = ar[i];
        }
    }
    return mx;
}

Min Element — TC: O(N), SC: O(1)

#include <climits>

int minArray(int ar[], int N){
    int mn = INT_MAX;
    for(int i=0; i<N; i++){
        if(ar[i] < mn){
            mn = ar[i];
        }
    }
    return mn;
}

Sum — TC: O(N), SC: O(1)

int sum(int ar[], int N){
    int s = 0;
    for(int i=0; i<N; i++){
        s = s + ar[i];
    }
    return s;
}

Frequency of k — TC: O(N), SC: O(1)

int frequency(int ar[], int N, int k){
    int c = 0;
    for(int i=0; i<N; i++){
        if(ar[i] == k){
            c++;
        }
    }
    return c;
}

Search k — Linear Search

bool searchK(int ar[], int N, int k){
    for(int i=0; i<N; i++){
        if(ar[i] == k) return true;
    }
    return false;
}
// TC: O(N)  SC: O(1)
07 Check Increasing / Strictly Increasing Compare adjacent · bool return ▾

Definitions

Increasing Order: Value can either increase OR remain the same (ar[i] ≥ ar[i-1])

Strictly Increasing: Value must always increase — no repeats allowed (ar[i] > ar[i-1])

Idea: For every element, compare with its previous. If any ar[i] ≤ ar[i-1] (for strict), return false.

bool isIncreasing(int ar[], int N){  // TC:O(N) SC:O(1)
    for(int i=1; i<N; i++){   // i starts from 1!
        if(ar[i] < ar[i-1]){  // prev > curr → not increasing
            return false;
        }
    }
    return true;
}
// For vectors — same logic with v.size()
bool isIncreasing(vector<int> v){
    int N = v.size();
    for(int i=1; i<N; i++){
        if(v[i] <= v[i-1]){  // strictly: use <= here
            return false;
        }
    }
    return true;
}

Problem

Given vector v, l and r: reverse vector from index l to r and return it.

Constraints: 0 ≤ l ≤ r < N, 1 ≤ N ≤ 10⁵

Example: v[8] = {10,20,25,30,12,14,16,29}, l=2, r=5

After reversing [l..r]: {10, 20, 14, 12, 30, 25, 16, 29}

Idea

Swap 1st element with last, 2nd with 2nd-last, 3rd with 3rd-last...

Use two pointers: l++ and r-- until l > r

// Iterative — TC: O(N), SC: O(1)
vector<int> reverse(vector<int> v, int l, int r){
    while(l <= r){
        int t = v[l];
        v[l] = v[r];
        v[r] = t;
        l++;
        r--;
    }
    return v;
}
// Call: reverse(v, 0, N-1) to reverse entire vector

12 Rotate Array Right-to-Left Once Store last · shift right · place at 0 ▾

Rotation (Right to Left)

Last element moves to index 0. All others shift one position right.

{10, 20, 30, 40, 50} → {50, 10, 20, 30, 40}

// TC: O(N), SC: O(1)
vector<int> rotate(vector<int> v){
    int N = v.size();
    int t = v[N-1]; // store last element
    for(int i=N-1; i>0; i--){  // i: [N-1..1]
        v[i] = v[i-1]; // shift right
    }
    v[0] = t; // place saved element at front
    return v;
}
13 2nd Smallest Element Track min & second min ▾

Given v[9] = {4, 6, 7, 2, 9, 3, 10, 5, 12}

Idea

Track two variables: min1 (smallest) and min2 (2nd smallest).

For each element: if smaller than min1, update both. If between min1 and min2, update min2.

int secondSmallest(vector<int> v){
    int N = v.size();
    int min1 = INT_MAX, min2 = INT_MAX;
    for(int i=0; i<N; i++){
        if(v[i] < min1){
            min2 = min1;
            min1 = v[i];
        } else if(v[i] < min2 && v[i] != min1){
            min2 = v[i];
        }
    }
    return min2;
}
// TC: O(N), SC: O(1)
Class 14 — Rotate Array k Times 8 pages
14 Rotate k Times — Brute Force O(N×k) · Nested loops ▾

Problem

Given vector v[N] and k, rotate it k times from right to left.

Example: v=[10,20,30,40,50,60,70], k=3 → [50,60,70,10,20,30,40]

Brute Force Idea: Loop k times, each time rotate once.

// Brute Force — TC: O(N×k), SC: O(1)
vector<int> rotateK(vector<int> v, int k){
    int N = v.size();
    for(int l=1; l<=k; l++){  // outer loop: k times
        int t = v[N-1];
        for(int i=N-1; i>0; i--){  // inner: N-1 iters
            v[i] = v[i-1];
        }
        v[0] = t;
    }
    return v;
}
// Total = k × (N-1) = O(kN) — TLE if N=k=10^5!
15 Rotate k Times — Optimized (3-Step Reverse) O(N) · Observation-based · Elegant ▾

Key Observation

After rotating k times, the last k elements move to start and first N-k elements move to end.

v[N] with k rotations → [v[N-k]..v[N-1]] + [v[0]..v[N-k-1]]

3-Step Algorithm — TC: O(N), SC: O(1)

Step 1: Reverse ENTIRE array (indices 0..N-1)

Step 2: Reverse FIRST k elements (indices 0..k-1)

Step 3: Reverse REMAINING elements (indices k..N-1)

vector<int> rotateKOpt(vector<int> v, int k){
    int N = v.size();
    reverse(v, 0, N-1);    // Step 1: reverse all
    reverse(v, 0, k-1);    // Step 2: reverse first k
    reverse(v, k, N-1);    // Step 3: reverse rest
    return v;
}

Class 15 — Pass by Value vs Reference 10 pages
16 Pass by Value vs Pass by Reference Memory · Stack · Heap · Arrays always PBR ▾

Pass by Value

A COPY of the value is made. Modifying in function does NOT affect original.

Used for: int, float, char, double (primitives)

Pass by Reference

The function receives a POINTER to the same memory. Modifications affect original!

Used for: arrays (always PBR in C++)

void changed(int cp[]){  // cp is a pointer to ar's memory
    for(int i=0; i<5; i++){
        cp[i] = cp[i] + 1;
    }
}

int main(){
    int ar[5]; ar[0]=10; ar[1]=20; ar[2]=30; ar[3]=40; ar[4]=50;
    changed(ar);  // modifies the ORIGINAL ar!
    for(int i=0; i<5; i++) cout << ar[i] << " ";
    // Output: 11 21 31 41 51
}

Analogy

Pass by Value: Photocopy of a document — writing on it doesn't affect the original.

Pass by Reference: Another NAME for the same box — "ar" and "cp" refer to the same memory.

TC: O(1) SC: O(1) Arrays always PBR

17 Elements Greater Than Itself Nested loops · O(N²) · Count ▾

Problem (Class 16)

Given ar[N]: return count of elements where there exists at least one element greater than ar[i] in the array.

Example: ar = {7,3,10,8,9,10,6} → ans = 5 (10 and 10 have none, 3,6,7,8,9 each have something greater)

Idea

For every ar[i], iterate and check if any element > ar[i] exists. If yes, c++.

int countGTSelf(int ar[], int N){
    int c = 0;
    for(int i=0; i<N; i++){
        for(int j=0; j<N; j++){
            if(ar[j] > ar[i]){
                c++;
                break; // found one, move to next i
            }
        }
    }
    return c;
}
// TC: O(N²), SC: O(1)

Optimization Hint

Find the max of the array. Count of elements NOT equal to max = answer. TC: O(N)

18 Count Unique Elements Sorted · Adjacent check · O(N) ▾

Problem (Class 16)

Count the number of unique/distinct elements in the array.

// After sorting — TC: O(N log N), SC: O(1)
int countUnique(vector<int> v){
    int N = v.size();
    sort(v.begin(), v.end());
    int cnt = 1;
    for(int i=1; i<N; i++){
        if(v[i] != v[i-1]){
            cnt++;
        }
    }
    return cnt;
}

20 Count Triplets: ar[i] < ar[j] < ar[k] Triple nested loop · O(N³) · Class 17 ▾

Problem (Class 17)

Count triplets (i, j, k) where i < j < k and ar[i] < ar[j] < ar[k].

int countTriplets(int ar[], int N){
    int cnt = 0;
    for(int i=0; i<N; i++){
        for(int j=i+1; j<N; j++){
            if(ar[j] > ar[i]){
                for(int k=j+1; k<N; k++){
                    if(ar[k] > ar[j]){
                        cnt++;
                    }
                }
            }
        }
    }
    return cnt;
}
// TC: O(N³), SC: O(1)

23 Transpose of Matrix Rectangular · Square · In-place · Class 19 ▾

What is Transpose?

Transpose of mat[i][j] = mat[j][i]. Rows become columns, columns become rows.

For N×M matrix → Transpose is M×N matrix.

Rectangular Matrix — TC: O(N×M), SC: O(N×M)

// Need new matrix for rectangular
vector<vector<int>> transpose(
    vector<vector<int>> mat, 
    int N, int M)
{
    vector<vector<int>> res(M, 
        vector<int>(N));
    for(int i=0; i<N; i++)
        for(int j=0; j<M; j++)
            res[j][i] = mat[i][j];
    return res;
}

Square Matrix — In-place TC: O(N²), SC: O(1)

// Only swap upper triangle!
void transposeSquare(
    vector<vector<int>>& mat, int N)
{
    for(int i=0; i<N; i++){
        for(int j=i+1; j<N; j++){
            swap(mat[i][j], mat[j][i]);
        }
    }
}
24 Rotate Matrix by 90° Transpose + Reverse rows · O(N²) ▾

Algorithm (Class 19)

Step 1: Transpose the square matrix (swap mat[i][j] with mat[j][i])

Step 2: Reverse each row

Result = 90° clockwise rotation!

// Rotate 90° clockwise — TC: O(N²), SC: O(1)
void rotate90(vector<vector<int>>& mat, int N){
    // Step 1: Transpose
    for(int i=0; i<N; i++)
        for(int j=i+1; j<N; j++)
            swap(mat[i][j], mat[j][i]);
    // Step 2: Reverse each row
    for(int i=0; i<N; i++)
        reverse(mat[i].begin(), mat[i].end());
}
